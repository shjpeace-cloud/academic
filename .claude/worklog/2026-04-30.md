### 5. 자동 push 셋업 — Stop 훅 (사용자 요청)

- **문제**: 사용자가 "왜 안바뀌는지 알았다" — 변경사항이 commit/push 되지 않아 사이트에 반영되지 않았던 것. Claude는 로컬 파일만 수정하고 자동으로 push하지 않음.
- **해결**: `.claude/settings.json` 에 Stop 훅 추가. Claude가 응답을 마칠 때마다 `bash .claude/auto-push.sh` 실행.
- **`.claude/auto-push.sh`** 동작:
  1. `git status --porcelain` 으로 변경 감지 → 없으면 조용히 종료
  2. `git add -A` → 추적/미추적 파일 모두 스테이징
  3. `git commit -m "auto: changes from Claude Code session"` (Co-Authored-By: Claude trailer 포함)
  4. `git push origin main`
  5. 결과를 systemMessage JSON으로 출력 → Claude Code UI에 "auto-pushed <SHA> to origin/main." 표시
  6. 실패 시 (push 거부, 충돌 등) 사용자에게 "수동으로 git push 하세요" 안내
- **장점**: 의미 단위로 별도 commit 메시지를 쓰지 않음 → 매 turn 하나의 자동 commit. 메시지가 generic해서 worklog/PR description으로 보충해야 함.
- **단점/주의**: 
  - 의도치 않은 변경도 push됨 (특히 wrong branch나 실험 중일 때 주의)
  - 다른 브랜치에서 작업 중이면 push가 main으로 안 됨 → 메시지로 안내됨
  - Stop 훅은 Claude Code 세션이 끝날 때마다 발동되므로, 의식적으로 commit을 나누고 싶으면 일시 비활성화 (`/hooks` UI) 필요

### 6. 오늘의 commit 분리 (자동 push 발동 후)

- 오늘 작업한 것들은 자동 push 훅이 작동하면 한 commit으로 묶일 가능성 높음. 메시지가 "auto:"로 시작하므로 의미 있는 커밋 메시지가 필요하면 `git commit --amend` 또는 별도 PR로 정리할 것.
